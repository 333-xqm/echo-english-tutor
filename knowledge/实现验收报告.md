## 关键问题汇总（按修复优先级）

### 🔴 阻断级（P0 未完成，用户不可用）

1. **缺少做题入口** — etchQuiz() 定义了但无按钮调用。需在 quick-reply 或 input-area 区域加一个"做道题"按钮。
2. **addMessage 函数不存在** — 第 412 行 ddMessage() 应改为 ddMsg()，否则调用时直接报错。
3. **checkAnswer 无判断逻辑** — 始终发 correct: true，不做真实对比。应向服务器传 user_label 让后端判断，或前端直接用 question_id 查本地答案。
4. **答题卡无反馈区域** — etchQuiz() 生成的卡片缺少 .quiz-fb 反馈 div，用户看不到对错原因。

### 🟡 严重级（功能可用但体验残缺）

5. **stats-bar DOM 缺失** — 统计栏只在 CSS 中存在，body 中没有 <div class="stats-bar">，efreshStats() 写入不存在的元素。需在 progress-bar 与 chat-area 之间插入统计栏。
6. **自适应升级/降级的 mastery 计算缺陷** — 新阶段 count_total=0 时 mastery 被错误计算为 0，导致 stage>1 无条件降级。应在 count_total < 3 时维持当前阶段。
7. **题库太浅** — 仅 3 个阶段 6 道题，至少应补齐阶段 4、5，每阶段扩充到 10+ 题。

### 🟢 建议级（体验可优化）

8. **阶段圆点加 tooltip** — /api/stats 的 stage_mastery 数据已可用，给 progress-node 添加 title 或 hover 事件即可。
9. **进度条改为 API 驱动** — updateProgress() 不应依赖 AI 回复中的"阶段X"文本来匹配更新。
10. **错题 >= 3 优先同类题型** — 自适应规则的第三条未实现。
