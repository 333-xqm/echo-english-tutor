# Echo 英语教练网站 — 体验改进方案

> 基于「AI 10倍速学习法」理念，结合当前代码现状的完整改进建议

---

## 一、当前交互体验评估

### 现状概括

当前网站本质上是一个**带预设按钮的AI聊天界面**。前端通过 `/api/chat` 把用户输入发给后端，后端调用AI API（或本地模板），返回纯文本。五阶段进度条是视觉装饰——它靠正则匹配AI回复中的「阶段X」文本来被动更新，没有任何实际的状态管理。

### 按维度拆解

| 评估维度 | 当前状态 | 问题等级 |
|----------|---------|---------|
| **即时反馈** | 只有AI的文本回复，没有结构化对错判断。用户输入A/B/C，AI文本解析判断，纯文本聊天 | P0 |
| **自适应难度** | 完全无。AI每次对话独立，无用户正确率/薄弱点记录 | P0 |
| **进度可视化** | 5个阶段圆点静态装饰，updateProgress()只在收到含「阶段X」回复时触发 | P0 |
| **量化进步** | 完全缺失。无正确率、答题数、词汇量等统计数据 | P0 |
| **间隔复习** | 无。每次对话全新上下文（仅保留20轮历史） | P1 |
| **多模态输入** | 纯文本。无徽章、卡片式答题、进度动画 | P1 |
| **学习闭环** | 输入→AI回答 结束。没有「练习→反馈→调整」闭环 | P0 |

### 关键数据流断裂

- 后端 `/api/records` 和 `/api/save_record` 已存在，但前端**从未调用**
- SKILL.md 内置30+道分阶段题库和「连对3题晋级」闯关逻辑，前端完全无法利用——所有做题都在AI文本回复中发生
- 后端没有专门的知识点/错题/正确率统计接口

## 二、基于「10倍速学习法」的改进建议

### P0 — 必须做（核心学习体验）

---

#### P0-1：结构化练习卡片 + 即时反馈

**问题**：当前「出题→答题→判对错」全在Chat界面靠文本完成。用户输入「A」，AI回复「对了」或「错了」。无法自动化统计，也无法确认用户答了什么。

**方案**：新增答题卡片组件，将带答案格式的文本渲染成交互式选择题UI，点击即判对错，数据自动上报。

**修改文件**：`static/index.html` + `server.py`

#### P0-2：自适应出题系统

**问题**：AI每次对话从零开始，不知道用户错在哪里。SKILL.md里有完整的摸底测试→分阶段→闯关逻辑，但网页完全没用上。

**方案**：后端新增 `/api/quiz` 端点，根据用户历史正确率、错题分布自动选题。

**自适应规则**：
- 当前阶段正确率 < 40% → 降一级出题（巩固薄弱）
- 当前阶段正确率 > 80% → 升一级出题（挑战更高）
- 错题数 >= 3 → 优先出同类题型（针对性复习）

#### P0-3：进度可视化联动真实数据

**问题**：阶段圆点只是视觉装饰（灰色/绿色切换），不反映用户真实学习进度。

**方案**：后端 `/api/stats` 返回完整统计数据，前端联动更新。

**修改文件**：`static/index.html` + `server.py`

### P1 — 建议做（显著提升体验）

---

#### P1-1：间隔复习提醒

**问题**：没有复习机制。新知识覆盖旧知识，学了前面忘后面。

**方案**：后端 `/api/review` 返回待复习内容（基于错题时间和频率），前端在用户空闲时提示。

**后端逻辑**：按时间排序最近错误题目，如果错题总数 >= 3 则返回复习提醒。

**前端逻辑**：用户连续对话5轮后或页面空闲10分钟后自动调用 `/api/review`，有需要复习则显示提示消息。

#### P1-2：学习记录面板

**问题**：用户看不到自己的学习轨迹。今天学了什么，正确率趋势如何，一概不知。

**方案**：在header新增「学习记录」按钮，弹出底部面板展示：
- 四格统计卡片（总答题数/正确率/当前阶段/错题数）
- 各阶段掌握度进度条（5个横条+百分比）

**修改文件**：`static/index.html`

---

#### P1-3：学习计划引导

**问题**：SKILL.md 有详细的个性化学习计划制定流程，网页完全没用。

**方案**：欢迎卡片增加「制定学习计划」入口，AI根据用户输入的时间安排生成结构化计划并持久化。

## 六、关键设计原则

1. **不要依赖AI识别答题**：前端直接渲染选择题UI，点击即判对错，答案不经过AI层面。这是速度、准确率、数据可信度的底线。

2. **进度不停留在「阶段」粒度**：细化到每个阶段内子知识点掌握度（如 be动词80%，人称代词60%）。

3. **自适应最小可行版本**：P0阶段不必机器学习。简单规则即可——正确率<40%降级、>80%升级、错题>=3优先同类题。

4. **反馈速度是生命线**：点击选项后200ms内显示对错反馈。不等AI。正确答案和解释直接从题库匹配。
